# 智盈AI登录和部署故障排除指南

## 登录问题排查

### 问题1: 无法登录，提示账号或密码错误

**可能原因:**
- 本地存储中的用户数据未正确初始化
- 浏览器本地存储被禁用或损坏
- 测试账号输入错误

**解决方案:**
1. **使用正确的测试账号**
   - 用户名: admin
   - 密码: admin123

2. **初始化用户数据**
   - 打开浏览器开发者工具（F12）
   - 进入Console选项卡
   - 运行以下代码：
   
   ```javascript
   // 初始化用户数据
   localStorage.setItem('users', JSON.stringify([
     { username: 'admin', password: 'admin123', isAuthorized: true }
   ]));
   // 清除当前用户信息
   localStorage.removeItem('currentUser');
   // 刷新页面
   location.reload();
   ```

3. **检查浏览器设置**
   - 确保浏览器允许本地存储
   - 清除浏览器缓存和Cookie
   - 尝试使用不同的浏览器

### 问题2: 登录后页面空白或跳转失败

**可能原因:**
- 本地存储权限问题
- 前端路由配置错误
- 浏览器兼容性问题

**解决方案:**
1. **检查浏览器控制台错误**
   - 打开浏览器开发者工具（F12）
   - 查看Console选项卡中的错误信息
   - 根据错误信息进行修复

2. **检查本地存储**
   - 进入Application选项卡
   - 检查Local Storage中是否有用户数据
   - 确保currentUser和users数据存在

3. **尝试清除本地存储**
   ```javascript
   localStorage.clear();
   location.reload();
   ```

## 部署问题排查

### 问题1: 部署到GitHub Pages失败

**可能原因:**
- 文件路径过长（Windows系统限制）
- Git仓库权限问题
- 构建文件过大

**解决方案:**
1. **使用Vercel或Netlify部署**
   - 这两个平台对文件路径长度没有限制
   - 部署流程更简单

2. **优化构建文件**
   - 检查项目中是否有不必要的大文件
   - 优化图片和资源文件
   - 考虑使用代码分割减少文件大小

### 问题2: 部署后页面无法访问

**可能原因:**
- 构建失败
- 路由配置错误
- 资源路径错误

**解决方案:**
1. **检查构建状态**
   - 确保 `npm run build` 命令执行成功
   - 检查构建输出目录是否正确生成

2. **检查路由配置**
   - 确保Vite配置中的base路径设置正确
   - 检查Netlify和Vercel的重定向规则

3. **检查资源路径**
   - 确保所有资源文件路径正确
   - 检查index.html中的资源引用

### 问题3: 部署后数据不显示

**可能原因:**
- API访问限制
- CORS跨域问题
- 数据来源配置错误

**解决方案:**
1. **检查API访问**
   - 确保股票数据API可以正常访问
   - 检查浏览器控制台是否有API错误

2. **检查CORS设置**
   - 确保API支持跨域请求
   - 考虑使用代理服务器

3. **检查数据来源配置**
   - 确保数据来源配置正确
   - 尝试切换不同的数据来源

## 本地开发问题排查

### 问题1: 本地服务器无法启动

**可能原因:**
- 端口被占用
- 依赖安装不完整
- 配置文件错误

**解决方案:**
1. **检查端口占用**
   - 关闭占用端口的其他进程
   - 或修改Vite配置使用不同端口

2. **重新安装依赖**
   ```bash
   rm -rf node_modules
   npm install
   ```

3. **检查配置文件**
   - 确保vite.config.ts配置正确
   - 检查package.json脚本配置

### 问题2: 本地开发时登录失败

**可能原因:**
- 本地存储被清除
- 开发服务器重启导致状态丢失
- 代码修改导致登录逻辑错误

**解决方案:**
1. **重新初始化用户数据**
   - 按照前面的方法初始化用户数据

2. **检查登录逻辑**
   - 检查Login.tsx和App.tsx中的登录逻辑
   - 确保onLogin回调函数正确执行

3. **检查控制台错误**
   - 查看浏览器控制台是否有错误信息
   - 检查网络请求是否正常

**步骤:**
1. 打开浏览器开发者工具（F12）
2. 进入Application选项卡
3. 选择Local Storage
4. 检查是否有users和currentUser数据

**步骤:**
1. 运行 `npm run build`

4. 确保所有必要的文件都已构建

## 常见错误及解决方案

### 错误1: "localStorage is not defined"

**原因:** 代码在非浏览器环境中尝试访问localStorage
**解决方案:** 确保代码只在浏览器环境中访问localStorage，添加环境检查

### 错误2: "Cannot read property 'username' of null"

**原因:** currentUser为null但代码尝试访问其属性
**解决方案:** 在访问currentUser属性前添加null检查

### 错误3: "Failed to fetch data"

**原因:** API请求失败或数据来源配置错误
**解决方案:** 检查网络连接，尝试切换数据来源

### 错误4: "404 Not Found"

**原因:** 资源文件路径错误或路由配置错误
**解决方案:** 检查资源路径和路由配置
